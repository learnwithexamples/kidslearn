⚡ Big-O & Complexity

How we measure the speed of an algorithm

← Back to Algorithms

⏱️ Why measure speed?

Two programs can solve the same problem, but one might take a millisecond and the other a full minute. As the amount of data grows, that gap explodes. Big-O notation is the language we use to describe how an algorithm's work grows as the input gets bigger.

We don't measure in seconds (computers differ!). Instead we count how many steps the algorithm takes relative to the input size, which we call n.

🎯 O(1) — Constant time

The work is the same no matter how big the data is. Grabbing one item by its index is instant:

def first(items): return items[0] # one step, always

➡️ O(n) — Linear time

The work grows in direct proportion to the input. One loop = O(n):

def total(items): s = 0 for x in items: # runs n times s += x return s

Double the list, and it takes about double the steps.

🐢 O(n²) — Quadratic time

A loop inside a loop means n × n steps. This gets slow fast — 100 items = 10,000 steps!

def has_duplicate(items): for i in items: # n times for j in items: # n times each ... # n × n = n² steps

📏 The rules of Big-O

Simplify, simplify

  • Drop constants: O(2n) becomes O(n). We care about growth, not exact counts.
  • Drop smaller terms: O(n² + n) becomes O(n²). The biggest term dominates.
  • Worst case: Big-O usually describes the slowest possible run.

✅ What you learned

  • Big-O describes how work grows with input size n.
  • One loop → O(n). Nested loops → O(n²). No loop → O(1).
  • Halving the problem each step → O(log n).
  • Drop constants and smaller terms when simplifying.

Task 1: Write an O(n) Function

Easy

Finish total(items) so it loops once through the list and returns the sum. Calling it on [1,2,3,4,5] should print 15. (This is O(n) — a single loop.)


                

                

                    
def total(items):
    s = 0
    for x in items:
        s += x
    return s

print(total([1, 2, 3, 4, 5]))

Task 2: The O(n²) Duplicate Finder

                    Medium
                

                
Write has_duplicate(items) using nested loops that returns True if any value appears twice. Test on [3, 1, 4, 1] → should print True. (Compare each pair of different positions.)

                

                    
                    

                        
                        
                        
                        
                    

                    

                

                

                    
def has_duplicate(items):
    for i in range(len(items)):
        for j in range(i + 1, len(items)):
            if items[i] == items[j]:
                return True
    return False

print(has_duplicate([3, 1, 4, 1]))

Task 3: Make It Fast — O(n)

                    Tricky
                

                
The same duplicate check can be done in O(n) using a set! Finish has_dup_fast: loop once, remember seen values in a set, return True the moment you see a repeat. Test on [1, 2, 3] (no dupes → False), then print O(n) is faster.

                

                    
                    

                        
                        
                        
                        
                    

                    

                

                

                    
def has_dup_fast(items):
    seen = set()
    for x in items:
        if x in seen:
            return True
        seen.add(x)
    return False

print(has_dup_fast([1, 2, 3]))
print("O(n) is faster")
